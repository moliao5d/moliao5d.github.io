日志简化写法

2017-06-26
技术

文章目录

写日志时候总是定义个静态的字段,从工厂类中获得日志实例,这一行代码就和getter setter一样没有没有任何技术含量,怎么样才能避免这一行呢,获取日志实例时候要把当前类的class传入,sun.reflect.Reflection有一个静态方法可以获得调用这个方法的类class对象,利用这个来避免写这一行代码,定义一个日志工具类

public class Xlog {

    public static void debug(String s){
        Class caller = Reflection.getCallerClass();
        Logger logger = LoggerFactory.getLogger(caller);
        logger.debug(s);
    }
    public static void debug(String s,Object... objs){
        Class caller = Reflection.getCallerClass();
        Logger logger = LoggerFactory.getLogger(caller);
        logger.debug(s,objs);
    }

    public static void err(String s){
        Class caller = Reflection.getCallerClass();
        Logger logger = LoggerFactory.getLogger(caller);
        logger.error(s);
    }
}

写日志时候直接调用工具类的方法来写,这样就不用在每个类中都定义一个日志字段了,当然,后果就是你为此多写了一个类…… 前面说道,这一行代码就像getter setter一样,lombok这个神器将我们从getter setter中解放出来,其实lombok也有一系列日志的注解,使用lombok时候,你只需@log/@slf4j/@log4j就可以了
